# فارکس بات — ربات معاملاتی خودکار چندحسابی/چندنمادی

پیاده‌سازی استراتژی SuperTrend + EMA(50/200) + RSI(14) از کد Pine Script اولیه، به‌همراه
داشبورد وب، مدیریت ریسک، و اتصال به حساب MT5 آلپاری از طریق MetaApi.cloud.

## ⚠️ قبل از هر چیز

- این ربات با **پول واقعی** معامله می‌کند. حتماً ابتدا با یک **حساب دمو** در آلپاری تست کنید.
- هیچ استراتژی، دقت ۸۰-۹۰٪ تضمین‌شده ندارد؛ نتایج بازار زنده می‌تواند با بک‌تست متفاوت باشد.
- مسئولیت تنظیم درست پارامترهای ریسک (`RISK_PERCENT`, `MAX_DAILY_LOSS_PERCENT`) با شماست.

## مراحل راه‌اندازی

### ۱. ساخت حساب MetaApi و اتصال آلپاری
1. در [app.metaapi.cloud](https://metaapi.cloud) ثبت‌نام کنید (پلن رایگان برای شروع کافی است).
2. از بخش **Add account**، حساب MT5 آلپاری خود را اضافه کنید — نیاز به:
   - شماره حساب (Login) و پسورد **سرور معاملاتی** MT5 (نه پسورد سایت آلپاری)
   - نام سرور MT5 آلپاری (مثلاً `Alpari-MT5` — دقیق آن را از پنل کاربری آلپاری یا ایمیل افتتاح حساب بردارید)
3. صبر کنید تا وضعیت حساب `DEPLOYED` و `CONNECTED` شود.
4. از بخش **API Tokens** یک توکن جدید بسازید.
5. شناسه حساب (Account ID) و توکن را در فایل `.env` قرار دهید.

### ۲. نصب روی سرور اوبونتو
پوشه پروژه را روی سرور آپلود کنید (مثلاً با `scp` یا `git`) و اجرا کنید:

```bash
sudo bash install.sh
```

این اسکریپت: پایتون و pip را نصب می‌کند، پروژه را در `/opt/forex-bot` قرار می‌دهد، محیط مجازی
و وابستگی‌ها را نصب می‌کند، فایل `.env` را از نمونه می‌سازد، و ربات را به‌عنوان سرویس systemd
روی پورت **۸۹۹۹** بالا می‌آورد.

> اگر پروژه را در گیت‌هاب خودتان قرار دهید، نصب واقعاً با یک خط ممکن می‌شود:
> ```bash
> curl -fsSL https://raw.githubusercontent.com/<user>/<repo>/main/install.sh | sudo bash -s -- https://github.com/<user>/<repo>.git
> ```

### ۳. تنظیم رمز داشبورد
فقط کافیست `/opt/forex-bot/.env` را باز کنید و `DASHBOARD_PASSWORD` را عوض کنید، بعد سرویس را ری‌استارت کنید:
```bash
sudo systemctl restart forex-bot
```
دیگر نیازی به تنظیم دستی توکن/حساب در `.env` نیست — همه از داخل داشبورد اضافه می‌شوند.

### ۴. افزودن حساب‌ها و جفت‌ارزها از داخل داشبورد
آدرس `http://IP-سرور-شما:8999` را باز کنید و با `DASHBOARD_USER` / `DASHBOARD_PASSWORD` وارد شوید.

- روی **«+ افزودن حساب جدید»** بزنید، نام، توکن MetaApi، شناسه حساب (Account ID)، حالت (`paper`/`live`) و
  درصد ریسک را وارد کنید و ذخیره کنید. می‌توانید چند حساب مختلف اضافه کنید — هرکدام کاملاً مستقل اجرا می‌شوند.
- داخل هر کارت حساب، روی **«+ افزودن جفت‌ارز»** بزنید و نماد (`EURUSD`, `GBPUSD`, `XAUUSD`, ...)، تایم‌فریم
  و پارامترهای استراتژی آن جفت‌ارز را مشخص کنید. هر جفت‌ارز چرخه بررسی سیگنال مستقل خودش را دارد و
  می‌توانید هم‌زمان چند جفت‌ارز را روی یک حساب فعال کنید.
- دکمه «شروع» روی هر حساب، همه جفت‌ارزهای فعالِ همان حساب را همزمان اجرا می‌کند. دکمه‌های «شروع همه» /
  «توقف همه» بالای صفحه روی همه حساب‌ها اثر می‌گذارند.

> **نکته درباره pip value:** برای جفت‌ارزهایی که یکی از طرفینشان USD نیست (مثل `EURGBP`)، محاسبه خودکار
> ارزش پیپ فقط تقریبی است. برای این‌گونه نمادها بهتر است مقدار «ارزش هر پیپ» را در فرم افزودن جفت‌ارز دستی وارد کنید.

> **مهاجرت از نسخه قبلی تک‌حسابی:** اگر قبلاً `.env` را با `METAAPI_TOKEN` و `METAAPI_ACCOUNT_ID` پر کرده
> بودید، اولین بار که سرویس بالا می‌آید آن تنظیمات به‌طور خودکار به یک حساب در `config/accounts.json` منتقل می‌شود.

## ساختار پروژه
```
app/
  main.py              سرور FastAPI + API کامل مدیریت حساب‌ها/جفت‌ارزها
  config.py            خواندن تنظیمات عمومی (پورت داشبورد، رمز) از .env
  core/
    strategy.py        پیاده‌سازی پایتونی SuperTrend + EMA + RSI (پارامتری، هر جفت‌ارز مستقل)
    risk.py             محاسبه حجم معامله + pip size/value برای چند نوع جفت‌ارز
    broker.py           اتصال به MetaApi (چند نمونه مستقل، یکی به ازای هر حساب)
    engine.py           AccountManager (هر حساب) + BotManager (نگه‌داری همه حساب‌ها)
    config_store.py     خواندن/نوشتن config/accounts.json + مهاجرت خودکار از .env قدیمی
  templates/dashboard.html
config/
  accounts.example.json  نمونه ساختار چندحسابی/چندنمادی
requirements.txt
install.sh
forex-bot.service
.env.example
```

## معماری چندحسابی/چندنمادی
- تنظیمات هر حساب و جفت‌ارزهایش در `config/accounts.json` نگه‌داری می‌شود (نه در `.env`).
- هر حساب یک اتصال مستقل به MetaApi دارد؛ اتصال یک حساب هیچ اثری روی حساب‌های دیگر ندارد.
- داخل هر حساب، هر جفت‌ارز فعال یک تسک async جدا و مستقل اجرا می‌کند — یعنی مثلاً EURUSD روی تایم‌فریم ۱
  ساعته و GBPUSD روی ۴ ساعته می‌توانند هم‌زمان و با پارامترهای متفاوت روی یک حساب اجرا شوند.
- سقف پوزیشن باز و سقف ضرر روزانه در سطح **هر حساب** تعریف می‌شود (نه در کل ربات)، چون این محدودیت‌ها
  معنایی مالی‌اند و باید نسبت به موجودی همان حساب سنجیده شوند.

## نکات امنیتی
- حتماً `DASHBOARD_PASSWORD` را تغییر دهید — بدون آن هر کسی با دسترسی به IP سرور می‌تواند ربات را کنترل کند.
- توصیه می‌شود پورت ۸۹۹۹ را فقط از طریق VPN یا با یک ریورس‌پراکسی (Nginx + HTTPS + IP whitelist) در دسترس بگذارید، نه مستقیم روی اینترنت باز.
- توکن MetaApi را در هیچ ریپازیتوری عمومی قرار ندهید.

## عیب‌یابی
- `journalctl -u forex-bot -f` برای مشاهده لاگ زنده سرویس.
- اگر پیام «METAAPI_TOKEN تنظیم نشده» دیدید، فایل `.env` را بررسی و سرویس را ری‌استارت کنید.
- اگر حساب در MetaApi وضعیت `DISCONNECTED` دارد، اتصال اینترنت ترمینال MT5 آلپاری یا صحت اطلاعات سرور را بررسی کنید.
